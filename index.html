浏览器

缓存

  • 缓存介绍

    缓存分为强缓存和协商缓存,流程图

    缓存机制相关的字段都是在请求和响应头上

    请求头:

  • 强缓存

    强缓存，在缓存有效期内，客户端直接读取本地资源

    强缓存返回状态码是200

    • Expries

      在http1.0 中使用，表示资源失效的具体时间点 Expires:Sat, 09 Jun 2018 08:13:56 GMT。若访问器和本地时间不一样，在现在的http 1.1 中换成了max-age

    • Cache-contol

      指定指令来是先缓存

      • max-age: 强缓存的有效时间，单位秒max-age=30672000
      • no-cache: 不使用本地缓存。需要使用缓存协商，先于服务器确认返回的响应是否被，如果之前的响应中存在ETag，那么请求的时候会与服务端验证，如果未被更改，则可以避免重新你下载
      • no-store: 直接禁止浏览器缓存数据，每次用户请求该资源，都会向服务端发送一个 请求，每次都会下载完整的资源
      • public: 可以被所有的用户缓存，包括用户和CDN等中间代理服务器
      • private: 只能被终端用户的浏览器缓存，不允许CDN等中继缓存服务器对其缓存

  • 协商缓存

    • 协商缓存，在使用本地缓存之前，需要先跟服务器做个对比，服务器告知你的资源可用，就可以直接使用本地资源。反之，服务器返回最新的资源给客户端，客户端收到后会更新本地资源

      • 状态码：
        • 若本地资源是最新的，返回304.
        • 重新获取资源，就正常返回200

    • Last-modified If-Modified-Since

      采用资源最后修改时间来判断，单位精确到秒

      Last-Modified: 服务器资源的最新更新时间Tue,14 Jan 2020 09:18:29 GMT

      If-Modified-Since:发起协商，把本地的记录文件更新时间传给服务器，服务器进行判断比较

      这个判断方式是http 1.0的产物,精确到秒，若文件的更新频率在秒级以内，就会出现文件不一致

    • ETag If-None-Match

      为了解决上面的问题，http 1.1加了这组标记

      ETage: 服务器根据内容生成唯一的字符串标识

      If-None-Match: 发起协商，把本地的hash标识传给服务器，服务器进行判断比较

  • 浏览器缓存存放位置

    缓存的两个来源：

    • memory cache:内存中读取
    • disk cache: 硬盘中读取

Map

含义和基本用法

JavaScript 的对象,本质上是键值对的集合 (Hash 结构) ,传统的只能用字符串当作键,有很大的限制性.Map数据解决了这个问题,使得”键”的范围不限于字符串,各种类型的值都可以当作键. 也就是Object结构提供”字符串-值”的对应,map数据提供”值-值”的对应

  • Map 构造函数接受数组作为参数.事实上任何据有Iterator接口的,且每个成员都是一个双元素的数组的数据结构都可以当作Map数据的参数

    • 如果对同一个键多次赋值,后面的值将覆盖前面的值

    • 如果读取一个未知的键,则返回的undefined

    • 注意: 只有对同一个对象的引用,Map数据才将视为同一个键.

实例属性和操作方法

  • 属性
    • size 返回Map数据成员的总数
    • Map.prototype.set(key,value)
    • Map.prototype.get(key) get 方法读取key 对应的键值.如果找不到key ,返回undefined
    • Map.prototype.has(key) 返回一个布尔值
    • Map.prototype.delete(key) 返回一个布尔值
    • Map.prototype.clearn() 没有返回值
  • 遍历方法
    • Map.prototype.keys()：返回键名的遍历器。
    • Map.prototype.values()：返回键值的遍历器。
    • Map.prototype.entries()：返回所有成员的遍历器。
    • Map.prototype.forEach()：遍历 Map 的所有成员。

Iterator 和for…of循环

遍历器(Iterator) 提供一个接口,为各种不同数据结构提供统一的访问机制.任何数据结构只要部署 Iterator 接口,就可以完成遍历操作.

Iterator

Iterator(遍历器)的概念

作用:

  • 为各种数据提供统一的,简便的访问接口
  • 使数据结构的成员能够按照某种依次排序
  • ES6创建一种新的遍历命令for....of 循环

Iterator 的遍历过程:

（1）创建一个指针对象，指向当前数据结构的起始位置。也就是说，遍历器对象本质上，就是一个指针对象。

（2）第一次调用指针对象的next方法，可以将指针指向数据结构的第一个成员。

（3）第二次调用指针对象的next方法，指针就指向数据结构的第二个成员。

（4）不断调用指针对象的next方法，直到它指向数据结构的结束位置。

每一次调用next方法，都会返回数据结构的当前成员的信息。具体来说，就是返回一个包含value和done两个属性的对象。其中，value属性是当前成员的值，done属性是一个布尔值，表示遍历是否结束。

默认Iterator接口

ES6 规定,默认的Iterator 接口部署在数据结构的Symbol.iterator 属性,只要数据结构具有Symbol.iterator 属性,就可以认为是’’可遍历的’’.

  • 原生具有Iterator 接口的数据结构如下:

    • Array
    • Map
    • Set
    • String
    • TypedArray
    • 函数的 arguments 对象
    • NodeList 对象

  • 对象如果想要具备可被for...of 循环调用的Iterator 接口,就必须在Symbol.iterator 的属性上部署遍历器的生成方法

    • 为一个对象添加Iterator 接口

调用 Iterator 接口的场合

在一些场景中会默认调用Iterator 接口

  • 解构赋值
  • 扩展运算符
  • yield*
  • 其他场合
    • for…of
    • Array.from()
    • Map(), Set(), WeakMap(), WeakSet()（比如new Map([['a',1],['b',2]])）
    • Promise.all()
    • Promise.race()

字符串的Iterator接口

字符串是一个类似数组的对象,也具有Iterator接口

Iterator接口与Generator 函数

Symbol.iterator 最简单的实现,还是使用Generator函数

for...of循环

只要数据结构部署Symbol.iterator 属性,就可以用for...of 循环遍历它的成员

数组

  • 数组原生具备Iterator 接口,for...of 循环本质上就是调用这个接口产生的遍历器

Set 和 Map 结构

计算生成的数据结构

有些数据结构是在现有的数据结构的基础上,计算生成的.比如，ES6 的数组、Set、Map 都部署了以下三个方法，调用后都返回遍历器对象。

  • entries() 返回一个遍历器对象，用来遍历[键名, 键值]组成的数组。对于数组，键名就是索引值；对于 Set，键名与键值相同。Map 结构的 Iterator 接口，默认就是调用entries方法。
  • keys() 返回一个遍历器对象，用来遍历所有的键名。
  • values() 返回一个遍历器对象，用来遍历所有的键值。

类似于数组的对象

  • for...of 循环能正确识别32为的UTF_16字符

  • 并不是所有的类似于数组的对象都具有Iterator 接口,解决办法就是使用Array.from 方法转为数组

对象

对于普通对象,for...of 结构不能直接使用,必须要部署了Iterator 接口后才能使用.但是,在这种情况下,for...in循环依然可以用来遍历键名

解决办法:

  • 使用Object.keys 方法将对象的键名生成一个数组,然后遍历这个数组

  • 另一个方法是使用Generator函数包装对象

与其他遍历语法的对比

  • for 循环

    • 写法比较麻烦

  • 数组内置的forEach

    • 无法中途跳出forEach循环,break和return命令都不行

  • for...in 可以遍历数组的键名

    • 数组的键名是数字.但是for...in 循环是以字符串作为键名’’0’’,”1”,”2”
    • for...in 循环不仅遍历数字键名还会遍历添加其他键,包括原型链上的键
    • 在某些情况下,for...in 循环会以任意顺序遍历键名
    • for...in 循环主要是为遍历对象而设计的,不适用于遍历数组

  • for...of循环的优点

    • 语法简洁
    • 可以与break.continue和return配合使用
    • 提供遍历所有数据接口的统一操作接口

  • 使用观察者模式

    不必手动运行整个架构

    package.json:

    ‘’watch’’:”webpack –watch”

  • 使用webpack-dev-server

    webpack-dev-server 提供一个简单的web服务器,实现实时重新加载

    • webpack.config.js

      devServer:{

      ​ contentBase:’./dist’

      }

      以上配置告知webpack-dev-server ,在localhost:8080 下建立服务器,将dist 目录下的文件作为可访问文件

    • package.json

      ‘’start’’:”webpack-dev-server –open”

  • 使用webpack-dev-middleware

    可以把webpack 处理后的文件传递给一个服务器(server).webpack-dev-server在内部使用它,同时可以作为一个单独的包使用,以便进行更多自定义设置来实现更多的需求

    • webpack.config.js

      publicPath会在服务器脚本中用到publicPath:’./‘

    • 创建server.js文件

    • package.json

      “server”:”node server.js”
