使用source map

为了更容易追踪错误和警告,JavaScript 提供了source map 功能,将变异后的代码映射回原始源代码

  • webpack.config.js

    devtool:”inline-source-map”

选用一个开发工具

  • 使用观察者模式

    不必手动运行整个架构

    package.json:

    ‘’watch’’:”webpack –watch”

  • 使用webpack-dev-server

    webpack-dev-server 提供一个简单的web服务器,实现实时重新加载

    • webpack.config.js

      devServer:{

      ​ contentBase:’./dist’

      }

      以上配置告知webpack-dev-server ,在localhost:8080 下建立服务器,将dist 目录下的文件作为可访问文件

    • package.json

      ‘’start’’:”webpack-dev-server –open”

  • 使用webpack-dev-middleware

    可以把webpack 处理后的文件传递给一个服务器(server).webpack-dev-server在内部使用它,同时可以作为一个单独的包使用,以便进行更多自定义设置来实现更多的需求

    • webpack.config.js

      publicPath会在服务器脚本中用到publicPath:’./‘

    • 创建server.js文件

    • package.json

      “server”:”node server.js”

模块热替换

允许在运行时更新各种模块,而无需进行完全刷新

启用HMR

  • webpack.config.js

    添加NamedModulesPlugin 容易查看要修补(path)的依赖

  • index.js

    告诉webpack接收更新的模块

通过Node.js API

注意: 当使用webpack dev server 和Node.js API 时,不要将dev server 选项放在webpack(webpack config object)配置对象中.而是在创建选项的时候,将其作为第二个参数传递,例如

new WebpackDevServer(compiler,options)
